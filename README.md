# 꽃배달 이용 가이드 · Flower Delivery Usage Guide

> 꽃 주문이 처음인 분을 위한 실전 이용 가이드.
> **현재 버전 v4.0.0**

---

## 이 프로젝트는

온라인으로 꽃배달을 주문하려는데 무엇을 어떻게 골라야 할지 막막한 분을 위한 가이드입니다. 꽃집 운영·창업이 아니라 **"고객이 어떻게 주문하면 되는가"** 에 초점이 있습니다.

| 대상 | 진입점 | 특징 |
| --- | --- | --- |
| 사람 | `index.html` | 브라우저에서 열면 바로 가이드 |
| AI 에이전트 | [`AGENTS.md`](AGENTS.md) + [`system.json`](system.json) | 1–2 파일 read로 모든 쿼리 해결 |
| 기여자 | [`CLAUDE.md`](CLAUDE.md) | 3계층 작업 원칙 · 수정 워크플로 |

---

## 빠른 시작

```bash
start index.html         # Windows
open index.html          # macOS
```

> 콘텐츠를 `fetch`로 불러오므로 `file://`로 열면 동작하지 않습니다. 로컬 서버로 띄우세요:
> `powershell -File scripts/serve.ps1` 또는 `python -m http.server`

배포: **https://dohoon0505.github.io/flower_dictionary/**

## 12개 장 구성 (v4.0.0)

'무엇을 보낼지 → 어떻게 주문하고 받을지 → 받은 뒤'까지 주문 여정 순서 그대로 설계했습니다. 총 **45개 아티클**.

| # | 장 | 다루는 내용 |
|---|------|----------|
| 01 | 꽃배달이 처음이라면 | 전국꽃배달 구조 · 주문부터 도착까지 5단계 |
| 02 | 상품별 특성 알아보기 | 근조·축하화환·개업화분·동양란·서양란·꽃다발·꽃바구니 7종 |
| 03 | 상황별 상품 추천 | 장례·결혼·개업·이취임·출산·생일·칠순·병문안·집들이·졸업 등 11가지 |
| 04 | 꽃말과 색상의 의미 | 색상의 의미 · 주요 꽃 꽃말 사전 |
| 05 | 기념일 캘린더 | 12개월 캘린더 · 14일 데이 · 가정의 달 |
| 06 | 리본 · 카드 메시지 문구 | 리본 구성 · 축하/근조 문구 예시 |
| 07 | 주문하고 결제하기 | 주문 방법 · 필요 정보 · 결제 · 증빙 |
| 08 | 시간 맞춰 배송 받기 | 마감 · 지역 · 시간지정 · 장소별 주의 |
| 09 | 받은 꽃 오래 보기 | 생화 · 난 · 관엽식물 · 보존화 관리 |
| 10 | 취소 · 환불 · 품질 보상 | 취소 시점 · 하자 보상 · 사진 차이 |
| 11 | 자주 묻는 질문 50선 | 주문·배송·상품·화환·문구 Q&A |
| 12 | 꽃배달 용어 사전 | 절화·후다지·오아시스 등 용어 풀이 |

각 장은 학습 목표 + 여러 아티클로 구성됩니다.

## 주요 기능

- **통합 검색** — 상단 헤더의 `검색` 버튼 또는 단축키 `Cmd/Ctrl+K` / `/` 로 호출. 장·아티클·본문 텍스트까지 한 번에 찾아 바로 이동.
- **전체 내용 다운로드** — 홈의 `전체 내용 다운로드 (.md)` 버튼으로 12개 장 45개 아티클 전체를 정리된 Markdown 한 파일로 내려받기. (정확도 메타데이터는 미포함)
- **단계형 주문 레일** — '주문부터 도착까지 5단계'를 한눈에 보여주는 시그니처 안내 컴포넌트.
- **라이트 / 다크 모드** — 토큰 기반 컬러 스왑. '따뜻한 안내소' 팔레트.
- **큰글씨 모드** — `큰글씨` 토글로 폰트·여백·둥글기 일괄 1.2배 확대. (처음 주문하는 분, 어르신께 특히 유용)
- **모바일 반응형** — 360px viewport에서 가로 스크롤 0건 검증.
- **선호 영속화** — 테마·큰글씨 설정은 `localStorage` 에 저장됨.

## 정보 정확도 메타데이터 (소스 전용)

각 아티클·본문 블록에는 `accuracy`(0~100) 메타데이터가 붙어 있습니다. **화면에는 노출되지 않고 데이터(소스)에만 존재합니다.** `100`은 1차 자료(논문)에서 저자가 직접 작성·검증한 내용, 그 외는 리서치로 교차검증해 추정한 신뢰도입니다. 규칙은 [CLAUDE.md](CLAUDE.md), 검증·집계는 `node scripts/validate.mjs`.

## 폴더 구조

```
flower_dictionary/
├── index.html                  ← 운영 진본 (단일 진입점)
├── assets/
│   ├── css/main.css
│   └── js/main.js
│
├── system.json                 ← 루트 매니페스트 (chapters + releases)
├── analyses/                   ← 장(챕터) 콘텐츠 데이터
│   └── {chapter-id}/chapter.json
│
├── scripts/validate.mjs        ← 무결성 검증
├── scripts/serve.ps1           ← 로컬 미리보기 서버
├── AGENTS.md
├── CLAUDE.md
└── README.md
```

## 기여

1. [CLAUDE.md](CLAUDE.md) 의 3계층 작업 원칙 준수 (운영 소스 · 데이터 · 문서 동시 갱신)
2. 수정 후 `node scripts/validate.mjs` 실행
3. `index.html`을 브라우저에서 열어 검색·라이트/다크·큰글씨·모바일 동작 확인
